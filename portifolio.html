projeto calendario

Esse projeto se trata de um tipo de agenda web, onde o cliente pode marcar algum horário disponível e o adm pode gerenciar esse horário. Páginas de login e cadastro, essa aplicação web tem uma página de cadastro e duas de login, a de cadastro e onde o usuário pode criar sua conta, a primeira página de login e onde o usuário entra com sua conta e é redirecionado para a página principal, já a segunda página de login é a do adm, onde ele coloca seu nome e sua senha, que é colocada por mim mesmo diretamente no banco de dados, é é redirecionado para a página de administração de horários.
Já as páginas após o login o cliente é redirecionado para a página principal, onde ele vai escolher o dia e clicar nele, após clicar no dia irá aparecer os horários disponíveis para aquele dia, os que estiverem disponíveis terão um botão que o usuário clica e agende um horário, caso o horário já estiver ocupado o botão não estará disponível, quando o usuário estiver procurando um dia para marcar o horário e esse dia já tiver passado, ou for sábado e domingo o dia não estará disponível, no caso só aparece uma mensagem falando da indisponibilidade.
já a página principal do adm é um pouco diferente, ela consiste em uma estrutura bem parecida, porém as coisas mudam quando você vai na parte dos horários disponíveis, lá terá uma duas caixas, uma para colocar o nome do cliente que deseja o horário e seu número de celular, já que as fezes é necessário mudar o horário, entre outras coisas, além de ter o botão de deletar, que limpa o horário que o adm desmarcar e o deixa livre novamente.

projeto pagamento via pix

Este projeto é uma aplicação web desenvolvida para facilitar o gerenciamento de cobranças via Pix, oferecendo suporte a dois perfis de acesso: usuário e administrador. O administrador possui uma página de login exclusiva, sendo que suas credenciais são previamente registradas no banco de dados por mim, garantindo maior controle e segurança no acesso. Por s ua vez, o usuário também conta com sua própria página de login, porém sua conta é criada unicamente pelo administrador, assegurando que apenas usuários autorizados possam acessar o sistema. No ambiente da aplicação, o administrador tem a possibilidade de criar cobranças para qualquer usuário cadastrado, informando o valor diretamente no campo correspondente ao usuário selecionado. As cobranças são geradas exclusivamente através do sistema de pagamentos Pix, proporcionando agilidade e praticidade no processo. Além disso, o administrador pode gerar relatórios detalhados de cada cobrança, onde é possível visualizar dados como o valor cobrado, o status do pagamento, o tempo que levou para a cobrança ser quitada, além das datas de criação e pagamento, permitindo um acompanhamento completo das transações simuladas. Do lado do usuário, a aplicação oferece a funcionalidade de consultar cobranças vinculadas à sua conta, de forma simples e intuitiva, e também simular o pagamento dessas cobranças, possibilitando uma experiência de uso próxima à realidade de um sistema de pagamentos. É importante destacar que o projeto utiliza exclusivamente o ambiente sandbox do Banco do Brasil, o que significa que todas as operações de pagamento realizadas são apenas simulações, sem envolvimento de transações financeiras reais, servindo apenas para fins de testes e demonstrações do funcionamento do sistema.

projeto Frete

Este projeto é uma aplicação web desenvolvida para gerenciar de forma completa e integrada todo o processo de transporte de cargas e fretes, oferecendo recursos que vão desde o cadastro seguro de usuários até o acompanhamento detalhado das entregas realizadas. O sistema conta com autenticação robusta, onde o cadastro de novos usuários exige nome entre 8 e 20 caracteres, senha contendo obrigatoriamente letras, números e pontos, também entre 8 e 20 caracteres, além de um e-mail válido. Para garantir maior segurança e confiabilidade no acesso, é utilizada uma API de envio de e-mails para autenticar o endereço eletrônico do usuário, enviando uma confirmação no momento do cadastro. A plataforma permite o registro de caminhões com informações como placa, modelo, ano de fabricação, capacidade máxima de peso e status operacional (ativo, manutenção ou indisponível), assegurando um controle eficiente da frota. Também é possível cadastrar cargas e fretes contendo dados detalhados, como origem, destino, tipo de carga (granel, container, frigorificada, entre outros), peso ou volume, datas de coleta e de entrega prevista, valor do frete, identificação do cliente remetente, do cliente destinatário e forma de pagamento (à vista, boleto ou prazo). O cálculo do valor do frete é realizado com base em uma tabela de preços por peso e distância, contando com integração a APIs de rotas que permitem determinar a quilometragem real a partir do trajeto efetivo, e não apenas da diferença nominal entre os locais, garantindo mais precisão nos orçamentos. Além disso, o sistema mantém um histórico detalhado de todos os transportes, registrando qual caminhão foi utilizado, as cargas transportadas, o status da entrega (pendente, em trânsito ou entregue), a data da entrega e o motorista responsável, permitindo à empresa e aos clientes acompanhar de forma clara, organizada e segura cada etapa do processo logístico.
